# TWGT Code of Conduct

## Our Commitment

TWGT is committed to building an open, respectful, and collaborative engineering community.

We encourage constructive discussion, knowledge sharing, and professional conduct in all interactions.

---

# Our Standards

Examples of positive behaviour include:

- Treating everyone with respect.
- Sharing knowledge openly.
- Providing constructive feedback.
- Supporting collaborative problem solving.
- Respecting different viewpoints.
- Documenting work clearly.
- Maintaining high engineering standards.

---

# Unacceptable Behaviour

Examples include:

- Harassment or discrimination.
- Personal attacks.
- Offensive or abusive language.
- Publishing confidential information.
- Disruptive behaviour.
- Deliberate misinformation.
- Malicious activity.

---

# Engineering Culture

TWGT promotes:

- Documentation First
- Security by Design
- Open Standards
- Continuous Learning
- Sustainable Engineering
- Responsible AI
- Professional Collaboration

---

# Responsibilities

Project maintainers are responsible for:

- Maintaining a welcoming environment.
- Reviewing contributions fairly.
- Applying project standards consistently.
- Protecting the integrity of the platform.

Contributors are responsible for:

- Following repository guidelines.
- Respecting other contributors.
- Writing maintainable code.
- Improving documentation where appropriate.

---

# Scope

This Code of Conduct applies to:

- GitHub repositories
- Discussions
- Issues
- Pull Requests
- Documentation
- Community spaces
- Official TWGT communication channels

---

# Reporting Issues

If you experience or observe behaviour that violates this Code of Conduct, report it privately to the project maintainers with sufficient information to investigate.

---

# Our Goal

TWGT aims to foster an engineering community built on professionalism, integrity, transparency, and continuous improvement.
